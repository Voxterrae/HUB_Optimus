What it is

HUB_Optimus is a governed system for separating signal from noise in high-friction contexts. It supports better judgment through structured scenarios, traceable reasoning, and controlled execution.

What it is not

  • Not an authority.
  • Not a prediction engine.
  • Not a replacement for diplomacy.
  • Not a public runtime service yet.

Public site scope

huboptimus.dev serves the public landing page only. Runtime files, backend operations, and versioned materials remain in GitHub.

Source of truth

GitHub repository, Issues, Pull Requests, commits, CI, and versioned documentation are authoritative. Chat is advisory. Public presentation does not override repository state.

Run locally

The current public release is a Python simulator and Semantic Engine CLI, plus static documentation. The validated v2.3.4 smoke path is:

git clone --branch v2.3.4 https://github.com/Voxterrae/HUB_Optimus.git
cd HUB_Optimus
python3 -m pip install -r requirements-dev.txt
python3 run_scenario.py --scenario example_scenario.json --seed 42
python3 -m semantic_engine.cli analyze examples/semantic_engine/case_minimal.json

Expected simulator result: success, with the configured success criteria reached in round 2.
