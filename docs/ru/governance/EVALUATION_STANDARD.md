# HUB_Optimus — Estándar de Evaluación

## Propósito
Este documento define la estructura mínima para que una evaluación sea compatible con HUB_Optimus.

Estandariza *cómo* se expresa el análisis, no *qué* conclusiones se alcanzan.

## Regla Central
Una evaluación solo es válida si los supuestos, riesgos, incentivos y límites de verificación son explícitos.

Los supuestos ocultos invalidan la confianza.

## Secciones Obligatorias

### 1. Definición del Escenario
- Partes involucradas
- Horizonte temporal
- Alcance (qué se incluye / excluye)

### 2. Objetivos Declarados
- Objetivos de cada parte
- Deben señalarse objetivos incompatibles

### 3. Restricciones
- Restricciones legales
- Restricciones políticas
- Restricciones materiales u operativas

### 4. Estructura de Incentivos
- Incentivos a corto plazo
- Incentivos a largo plazo
- Riesgos de desalineación

### 5. Clasificación de la Capa de Confianza
- Clase de evidencia (A / B / C)
- Mecanismos de verificación
- Nivel de confianza estructural

### 6. Vectores de Riesgo
- Riesgo de escalada
- Riesgo de bloqueo (lock-in)
- Riesgo de reversibilidad
- Riesgo de precedentes

### 7. Modos de Fallo
- Cómo podría fallar el escenario
- Señales tempranas de alerta

### 8. Evaluación de Estabilidad
- Condiciones que mejoran la estabilidad
- Condiciones que incrementan la inestabilidad

### 9. Incógnitas Explícitas
- Incógnitas conocidas
- Áreas que requieren más datos

## Contenido Prohibido
Las evaluaciones NO deben:
- asignar culpa moral,
- declarar ganadores o perdedores,
- presentar predicciones como certeza,
- apelar a autoridad o urgencia.

## Neutralidad de Resultado
HUB_Optimus no recomienda acciones.
Aclara consecuencias y riesgos estructurales.

## Revisabilidad
Toda evaluación debe ser:
- reproducible,
- cuestionable sección por sección,
- mejorable por consenso.
